# biopoly — forward + inverse ML for compostable biopolymer formulation

An end-to-end MLOps demo: predict the
properties of a compostable biopolymer formulation, and — the real prize — **invert
it**: given a target spec, propose the formulation that hits it.

> ⚠️ **Runs entirely on SYNTHETIC data generated by this repo.** It contains **no
> company or proprietary data** and no real measured samples. The generator
> ([`chemistry.py`](src/biopoly/data/chemistry.py) + [`generate.py`](src/biopoly/data/generate.py))
> uses plausible mid-range figures from the public biopolymer literature so the demo
> has realistic, learnable structure — see [`DATA_CARD.md`](DATA_CARD.md). Results are
> illustrative of the *approach*, not of any product.

Why this shape: the target use case is *"tell us what the material needs to do"* →
*"we generate the formulation."* That's exactly the inverse-design loop below.

## Quickstart

```bash
uv sync                          # Python 3.12 env from the locked deps
uv run biopoly-generate          # write data/formulations.parquet (~2000 rows)
uv run biopoly-train             # train forward model, log to MLflow, register champion
uv run uvicorn biopoly.api.main:app --reload   # serve the API on :8000

# or the whole service in one line:
docker compose up --build        # API on http://localhost:8000/docs
```

Predict properties, then design a formulation to a target. Every endpoint but `/health`
is tenant-scoped and needs an `X-API-Key` header (demo key below; see
[`docs/MULTI_TENANCY.md`](docs/MULTI_TENANCY.md)):

```bash
curl -s localhost:8000/predict -H 'X-API-Key: demo-acme-key' \
  -H 'content-type: application/json' -d '{
  "frac": {"PLA": 0.8, "PBS": 0.2}, "additives": {"plasticizer": 0.05},
  "process_temp_c": 195, "process_time_min": 20}'

curl -s localhost:8000/design -H 'X-API-Key: demo-acme-key' \
  -H 'content-type: application/json' -d '{
  "target": {"tensile_strength_mpa": 50, "optical_clarity_pct": 80},
  "method": "bayesopt", "top_k": 3}'

curl -s localhost:8000/history -H 'X-API-Key: demo-acme-key'   # this tenant's runs only
```

## The problem

**Formulation** (polymer types + ratios over `{PLA, PHA, PBAT, PBS, TPS, PCL}`,
additives, processing temperature & time) **→ five properties**: tensile strength,
melt flow index, biodegradation@60d, water absorption, optical clarity.

## How the code is organised

| Concern | Where it lives |
|---|---|
| Data: missingness, protocol covariate | [`generate.py`](src/biopoly/data/generate.py), [`features.py`](src/biopoly/features.py) |
| Forward model — GBDT, native categorical/missing, feature importance | [`models/forward.py`](src/biopoly/models/forward.py) |
| Hyper-parameter search (Optuna) | [`models/train.py`](src/biopoly/models/train.py) `--hpo` |
| Evaluation — uncertainty, **conformal calibration (CQR)** + **per-variable tolerance** | [`models/metrics.py`](src/biopoly/models/metrics.py), [`models/conformal.py`](src/biopoly/models/conformal.py) |
| **Signal processing** — synthetic DSC thermograms → `scipy.signal` DSP → melt-peak features | [`signals.py`](src/biopoly/signals.py) |
| System — FastAPI, Pydantic, Docker, tracking, registry, drift, retrain, CI | [`api/`](src/biopoly/api), [`tracking/`](src/biopoly/tracking), [`models/registry.py`](src/biopoly/models/registry.py), [`monitoring/drift.py`](src/biopoly/monitoring/drift.py), [`scripts/retrain.py`](scripts/retrain.py) |
| Inverse design — baseline sampling → Bayesian optimisation | [`inverse/`](src/biopoly/inverse) |
| Polymer representation — descriptors now, learned embeddings as upgrade | [`chemistry.py`](src/biopoly/data/chemistry.py), [`docs/RESULTS.md`](docs/RESULTS.md) |

## Results (synthetic holdout, seed 42)

| target | R² | MAE | within-tolerance |
|---|---|---|---|
| tensile_strength_mpa | 0.93 | 2.1 MPa | 77% |
| melt_flow_index_g10min | 0.93 | 1.5 | 89% |
| biodegradation_60d_pct | 0.94 | 3.5 % | 91% |
| water_absorption_pct | 0.99 | 0.4 % | 95% |
| optical_clarity_pct | 0.94 | 3.0 % | 84% |

**mean R² ≈ 0.94.** Every prediction ships with a p10–p90 band (quantile regression),
**conformally calibrated** ([`conformal.py`](src/biopoly/models/conformal.py)). On a held-out
calibration split, CQR corrects the raw quantile bands — which *under-cover* badly (~0.55 against
a nominal 0.80) — back to ≈0.80 with a distribution-free, finite-sample guarantee, widening the
intervals only as much as the data demands. `biopoly-train` prints the raw→conformal coverage for
every target, and the API serves the calibrated bands.

Regenerate the full report (figures + [`docs/RESULTS.md`](docs/RESULTS.md)) with:
`uv run python scripts/report.py`.

## MLOps

- **Experiment tracking** behind a thin `ExperimentTracker` protocol — **MLflow** by
  default, **ClearML** swappable (`BIOPOLY_TRACKING_BACKEND=clearml`), no-op for CI.
- **Model registry** ([`registry.py`](src/biopoly/models/registry.py)) — versioned models
  with a champion pointer; `register_if_better` promotes only on improvement, and rollback
  is one call. (Maps to MLflow Model Registry / SageMaker Model Registry.)
- **Drift monitoring** ([`drift.py`](src/biopoly/monitoring/drift.py)) — KS + PSI; catches
  the built-in mid-2025 supplier-purity shift.
- **Retraining trigger** ([`retrain.py`](scripts/retrain.py)) — detect drift → retrain →
  validate → register-if-better. Wired into GitHub Actions (`workflow_dispatch`).
- **Reproducibility** — `uv` lockfile, pinned Python, pre-commit (ruff) + a conventional-commit
  hook, typed FastAPI with Pydantic validation.
- **Spiral-layered tests** — every test carries a `@pytest.mark.layer(1..9)` from foundations
  (schema/chemistry) up to the full system (inverse design/API). CI runs them in order and reports
  a **complexity frontier** — the highest contiguous layer that fully passes — so a failure tells you
  *how far up the complexity ladder the implementation is correct*, not just a pass count. See
  [`tests/conftest.py`](tests/conftest.py). Multi-tenant productionisation is sketched in
  [`docs/MULTI_TENANCY.md`](docs/MULTI_TENANCY.md).
